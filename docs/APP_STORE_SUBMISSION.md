## Step-by-Step Submission Plan

### Phase 1: Prepare Assets (Day 1-2)

1. **Generate App Icons**
   ```bash
   # Use script or tool to generate all icon sizes from SVG
   # Copy to ios/App/App/Assets.xcassets/AppIcon.appiconset/
   ```

2. **Capture Screenshots**
   - Run app in simulators at required sizes
   - Capture each screen
   - Add text overlays in design tool

3. **Create App Preview Video (Optional)**
   - 15-30 second demo
   - Show key features

### Phase 2: App Store Connect Setup (Day 2-3)

1. **Create App Record**
   - Log into App Store Connect
   - Click "+" to add new app
   - Select iOS platform
   - Enter bundle ID, name, SKU

2. **Enter App Information**
   - Privacy policy URL
   - Category selection
   - Content rating questionnaire
   - Age rating

3. **Enter Pricing**
   - Free
   - No in-app purchases (v1.0)

4. **Enter Version Information**
   - Screenshots (all required sizes)
   - App preview (optional)
   - Promotional text
   - Description
   - Keywords
   - Support URL
   - Marketing URL
   - What's New

### Phase 3: Build & Upload (Day 3-4)

1. **Archive Build**
   ```bash
   # In Xcode
   Product > Archive
   ```

2. **Upload to App Store Connect**
   - Use Xcode Organizer
   - Or use Transporter app

3. **Wait for Processing**
   - Usually 15-30 minutes
   - Check for any errors

4. **Select Build**
   - In App Store Connect
   - Add build to version

### Phase 4: Submit for Review (Day 4)

1. **Complete App Review Information**
   - Demo account (if needed)
   - Notes for reviewer
   - Contact information

2. **Submit for Review**
   - Click "Submit for Review"
   - Answer export compliance
   - Answer content rights
   - Answer advertising identifier

3. **Monitor Review Status**
   - Typically 24-48 hours
   - May receive questions
   - Address any rejections

### Phase 5: Release (Day 5-7)

1. **Review Approved**
   - Choose release option:
     - Manual release
     - Automatic release
     - Scheduled release

2. **Post-Launch**
   - Monitor crash reports
   - Respond to reviews
   - Plan updates

## Common Rejection Reasons & Prevention

| Issue | Prevention |
|-------|------------|
| Crashes | Test thoroughly on multiple devices |

| Placeholder content | Remove all lorem ipsum, test images |
| Privacy policy issues | Ensure policy matches data collection |
| Misleading metadata | Screenshots must match app |
| Login issues | Provide working demo account |
